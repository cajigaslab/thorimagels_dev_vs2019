These tools prepare the tdNWLdriver for submission to Microsoft Developer Portal for signing by Microsoft.
This is the ONLY way to properly install Windows 10 kernel drivers after build 1607.
The procedure is a bit convoluted but if not precisely done, the Microsoft cross-signing process will fail in cryptic fashion.

Scripts originally done with a Digicert 'SafeNet' USB hardware key - note that file paths and/or certificate stores must be correct and complete.
